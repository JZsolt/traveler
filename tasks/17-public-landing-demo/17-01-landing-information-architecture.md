# 17-01 — Landing Information Architecture

**Estimate:** 1-2 hours

## Goal

Define the content structure for the public landing page.

## Scope

- Hero
- How it works
- AI planning value
- Offline/PWA value
- Demo trip preview
- Login/register CTA

## Acceptance Criteria

- Landing outline is documented.
- Public visitor and logged-in user CTA behavior is defined.
- No real private trip data appears in the IA.
- Design follows existing design-system primitives.

## Review Checklist

- [ ] Does not become a generic marketing page.
- [ ] First viewport clearly explains Traveler.
- [ ] Logged-in state points to `/app/trips`.
- [ ] Admin is not mentioned.
