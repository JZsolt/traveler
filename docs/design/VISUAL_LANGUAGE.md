# Traveler Visual Language

## Core direction

Traveler should feel like a calm, premium digital travel journal.

Not:
- admin dashboard
- generic SaaS UI
- Dribbble gradient demo
- Material UI default app

Yes:
- clean
- readable
- mobile-first
- kind and friendly
- editorial
- easy to edit
- spacious but not empty

## Signature idea

Traveler is a living travel book.

The UI should feel closer to:
- Apple Journal
- Flighty
- Notion Mobile
- Polarsteps

The main visual language is:
- strong typography
- generous whitespace
- simple sections
- timeline-based itinerary
- very few colors
- very few shadows
- consistent Lucide icons

## Golden rules

1. Typography and spacing should do more work than cards.
2. Do not put every element inside a card.
3. Use cards only for grouped, interactive, or emphasized content.
4. Use dividers and rows for simple content.
5. Use one primary color per screen.
6. Use orange only for AI / important CTA.
7. AI should feel helpful, not loud.
8. Prefer calm editorial layout over dashboard layout.
9. Every screen should have one main action.
10. If something appears twice, make it a reusable component.

## Page hierarchy

Every page should follow this structure:

Header
Hero / title area
Primary content
Secondary sections
Bottom navigation / action

## Visual rhythm

Use vertical rhythm:

- 32px between major sections
- 24px between section title and content
- 16px between related elements
- 8px for small internal spacing

## Avoid

- random gradients
- emoji as icon system
- too many floating buttons
- too many shadows
- multiple accent colors
- over-rounded everything
- dense dashboard cards
- AI sparkles everywhere
